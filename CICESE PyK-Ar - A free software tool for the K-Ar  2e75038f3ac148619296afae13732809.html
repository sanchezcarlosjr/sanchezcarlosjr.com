Potassium-Argon dating Software

Abstract

A numerical method K-Ar tracer, a web platform to date volcanic rocks 5 million years from getting spectrometer data.

While a lot of laboratories use a software insider, reproducibly requires open-source software availability and verifiability plus interoperability thanks Web. Therefore, it’s introducing the ‘PyK-ar’ package for Python, an open-source software package and web platform that dates volcanic rocks, it can manage them using a graphical user interface. The latest version of the package can be downloaded from GitHub.

The dating used in the package is, with some adaptations, largely based on the method originally proposed by G. B. Dalrymple and M. A. Lanphere (1969).

  • The package ‘PyK-Ar’ for Python can date volcanic rocks.

Justificación

Se requiere reducir los datos adquiridos por espectrometría de masas desde una plataforma sencilla. El proceso de reducción de datos usado actualmente requiere asociar grupos de datos diversos y fácilmente provoca errores del operador. Nuestro software ayudará al investigador Basabe del CICESE y también a otros investigadores del mismo campo.

Estado del arte

El estado del arte es el método numérico descrito en [3] (K-Ar con trazador), además de un software anterior escrito en C por los investigadores de CICESE, tales como el proyecto RaDAn (mezcla de archivos), IsoCalc (procesamiento) [5]. Como bases de datos, una bitácora de resultados en Google Sheets y un guardado temporal de los archivos obtenidos por el espectrómetro.

Objetivo general

Crear una plataforma web que procesará datos geocronológicos de K-Ar con trazado.

Objetivos específicos

  • Unificar los 3 procesos: lectura y unión de los archivos, procesamiento y cálculo de las edades con sus incertidumbres.
  • Mejorar la experiencia de usuario: reducir al máximo las tareas manuales.
  • Garantizar la portabilidad gracias a la Web.
  • Asegurar la simplicidad del código.
  • Garantizar la correcta funcionalidad en los navegadores Chromium, Firefox y Safari.
  • Ofrecer una interfaz amigable para el usuario.
  • Entregar una versión estable el 6 de diciembre de 2021.

Hipótesis

Se espera aprender sobre K-Ar, métodos numéricos aplicados y la generación de conocimiento. Además, también se espera la apreciación mutua en relación empresa y cliente.

Proceso de desarrollo

Aplicar el Modelo General del Proceso de Desarrollo de Software, utilizando un número selecto de artefactos del Proceso Unificado de Racional (RUP) con el Lenguaje Unificado de Moldelado (UML), tales como:

  • Máquina cliente: Windows 7 y navegador Chromium. Además de consultas de otras máquinas cliente, como navegadores principales: Chromium, Firefox y Safari.
  • Método K-Ar : método de datación radiométrica que se utiliza en geología y arqueología para datar rocas o cenizas volcánicas, en general, las más antiguas.
  • Lenguaje de programación Python e interfaz gráfica Web: para el uso correcto de la aplicación de métodos numéricos y estadístico se basará en Python, haciendo una "mezcla" de lenguajes.
  • Máquina servidor: Linux, Python (lenguaje servidor donde se procesaran los datos, tomado de la entrevista), JavaScript (React), Firestore y Google Cloud Platform.

Requerimientos

Requisitos funcionales obtenidos desde la entrevista y la carpeta compartida.

  • No es un sistema con roles, por tanto, todos son usuarios en calidad de investigadores.
  • Iniciar sesión con correo electrónico o Google, para llevar control de los resultados.
  • Lectura de archivos desde un formulario web (múltiples archivos): blanco y real. La información deben guardarse en la base datos.

  • Procesar los archivos mediante el método numérico y tomando como base el código actual, después de la lectura y guardado de la información del punto anterior -automáticamente-.
  • Mostrar todos los experimentos con la siguiente info:
    • Nombre de la muestra
    • Edad de la roca con dos cifras significativas.
    • Desviación estándar con dos cifras signficativas.
    • Nombre de operador.
    • Fecha.

  • Acceder al histórico para posteriores consultas.

  • Lectura de archivos desde una API (múltiples archivos): blanco y real, con la finalidad de futuros proyectos sin la necesidad de usar una interfaz gráfica. La información deben guardarse en la base datos.
  • Subir automáticamente los archivos cada vez que haya un análisis: cuando se guarden en la carpeta asignada.

Calidad de software

Siguiendo el estándar ISO 9126 identificamos los siguientes seis atributos clave de calidad:

  • Funcionalidad: La interfaz web cumple con los requisitos previamente establecidos por el cliente, como lo son; inicio de sesión, lectura de archivos (blanco y real), usar el método numérico K-Ar, portabilidad y la presentación de resultados (edad de la roca y su desviación estándar).
  • Confiabilidad: Como su uso es mediante Internet está disponible en múltiples dispositivos y diversos navegadores web, haciendo que la tolerancia a fallos sea muy buena. Además, usando servicios como lo es Firebase tenemos la seguridad totalmente respaldada gracias a Google que esta detras, y claro, al tener el software mediante internet podemos recuperar los archivos rápidamente y tenerlos seguros.
  • Usabilidad: Al ser un software muy específico tiene la facilidad de ser sencilla su interfaz, ya que está pensada para que sea manejada por el investigador, teniendo los menús necesarios para que el operador pueda entenderlos.
  • Eficiencia: Al tener una buena gestión de almacenamiento y uso de recursos con la aplicación de Firebase estamos seguros que se obtiene una optimización de tiempo realmente buena.
  • Facilidad de recibir mantenimiento: El software cumple con los criterios marcados por Clean Code.
  • Portabilidad: Gracias a que el software fue desarrollado desde el principio mediante vía web, no se necesita un equipo robusto para poder trabajar con la aplicación.

interface Cycle {
    cycle: string;
    mass: string; // Grams
    peak: string; // M36, M38, M40
    measure: string;
    intensity: string; // Volts
    time: string; // Milliseconds
}

interface Experiment {
    cycles: Cycle[];
    analysis_date: string;
    spectrum: string;
    sample_id: string;
    type: string;
    spectrum_user_name: string;
    file_name: string;
}

    Cycle. Identificador incremental del ciclo.

    Mass.

    Peak. Isotopos de Argon: "M40" (Argon 40), "M38" (Argon 38), "M36" (Argon 36), en otro caso indefinido: "ZERO".

    Measure.

    Inty.

    Time.

    Analysis Date.

    Spectrum.

    Sample id.

    Type.
