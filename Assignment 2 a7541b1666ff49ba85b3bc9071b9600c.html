If a client and server are placed separately, you may find that network latency dominates all throughput. How can we tackle this problem?

We can tackle this problem with async communication where the client becomes a subscriber and the server becomes a publisher. So, it decreases the network latency because you are decreasing the throughput. Another way is a transparent location or access whereby the client or server has some replica —cache. Other common strategies are compressing the data (e.g. Brotli), using CDN, minimizing the operations in the server, reusing the same connection, and optimizing bottlenecks.

What is a three-tier client-server architecture?

It is one that organizes the system into 3 distributed tiers often the user interface or the presentation tier, the controllers or application tier, and the data tier or models such that the client requests reaches all of them in sequential order. The system designer can choose where the components are located; we have not obligated to separate them into three different hardware nodes.
