  • Access to shared Java objects can be serialized by declaring its methods as being synchronized. Is this enough to guarantee serialization (consistency) when such an object is replicated?

No, replicated variables can be in an inconsistent state when different operations occur at the same time.

  • Explain in your own words what the main reason is for actually considering weak consistency models.

Because the user demands better performance even though we lose consistency between replicas. Today, we have an offline-first paradigm, so it’s more important the user has their own data than consistency between nodes.

  • Explain how replication in DNS takes place, and why it actually works so well.

Since DNS assumes the mapping table doesn’t change frequently, it can do a typical caching procedure, it does the DNS lookup when it doesn’t have the path to the destination, and it saves the path, otherwise, the lookups are looked up cached results.

  • During the discussion of consistency models, we often referred to the contract between the software and the data store. Why is such a contract needed?

Because we separate the component's responsibilities transparently, so, for example, the data store can follow sequential consistency and the software perceive a strong model, therefore the overall performance system increases.

  • It is often argued that weak consistency models impose an extra burden on programmers. To what extent is this statement actually true?

It depends on how the programmers manage the shared data, they can protect it with, for example, transactions or revisions. Generally speaking, we expect strong consistency models such as sequential models.

  • Does totally ordered multicasting by means of a sequencer and for the sake of consistency in active replication, violate the end-to-end argument in system design?

Yes. Since it added an indirection layer because a sequencer coordinates and achieves consistency, this model violates the end-to-end argument.

  • What kind of consistency would you use to implement an electronic stock market? Explain your answer.

The electronic stock market needs the last information that is independent even though our system loose consistency. So, causal consistency is the best option to achieve this requirement.

  • Consider a personal mailbox for a mobile user, implemented as part of a wide-area distributed database. What kind of client-centric consistency would be most appropriate?

Because a personal mailbox needs to track local changes and ensure replicates to other clients, the best choice is the local-write protocol.

  • Describe a simple implementation of read-your-writes consistency for displaying Web pages that have just been updated.

Today, the browser can be a replica itself, so, the user reads what he or she writes immediately. When the master responds, we’re going to update the browser.

  • To make matters simple, we assumed that there were no write-write conflicts in Bayou. Of course, this is an unrealistic assumption. Explain how conflicts may happen.

Conflicts occur, for example, in an application about time or schedule where a lot of changes can be propagated to other replicas and overwrite local changes, it deals with unexpected changes.

  • When using a lease, is it necessary that the clocks of a client and the server, respectively, are tightly synchronized?

No. The client can request a new lease before the current one expires when it takes a view that is according to synchronization.

  • We have stated that totally ordered multicasting using Lamport’s logical clocks does not scale. Explain why.

Since the totally ordered multicasting requires all nodes to be running and be notified when some of them be slow or crash, the performance decreases when the number of servers grows.
